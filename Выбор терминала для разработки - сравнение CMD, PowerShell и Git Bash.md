## Решение задачи "Калькулятор"

1. **Бесконечный цикл** (`while (true)`) — программа работает до остановки пользователем.

2. **Ввод данных через Scanner**:
   - Два числа типа `double` (`d1`, `d2`)
   - Операция как строка (`operation`)
   - `sc.nextLine()` после `nextDouble()` очищает буфер от символа переноса строки

3. **Вычисление через if-else**:
   - Проверка операции (`equals()`)
   - Выполнение соответствующего действия: умножение (`*`), сложение (`+`), деление (`/`), вычитание (`-`)
   - Если операция не распознана — сообщение об ошибке

4. **Вывод результата** — результат выводится после вычисления.

**Особенности решения:**
- Использование `double` для работы с дробными числами
- Очистка буфера после `nextDouble()` перед `nextLine()`
- Обработка неизвестных операций
- Бесконечный цикл для повторных вычислений

Код работает корректно и выполняет основные арифметические операции.

## Как работает проверка:

1. **`hasNextDouble()`** — проверяет, является ли следующее значение числом, не считывая его.
2. **Если число корректное** — считываем его через `nextDouble()`.
3. **Если число некорректное** — выводим сообщение об ошибке, очищаем буфер через `nextLine()` и переходим к следующей итерации цикла (`continue`).

## Альтернативный способ (через try-catch):

Если хотите использовать обработку исключений, можно так:

```java
try {
    d1 = sc.nextDouble();
} catch (InputMismatchException e) {
    System.out.println("Ошибка! Введите корректное число.");
    sc.nextLine();
    continue;
}
```

Текущий вариант с `hasNextDouble()` предпочтительнее, так как проверяет данные до чтения и не требует обработки исключений.

Теперь калькулятор корректно обрабатывает некорректный ввод и просит ввести число заново.

## Где взять Git Bash

Git Bash обычно устанавливается вместе с Git for Windows:
1. Скачайте Git: https://git-scm.com/download/win
2. При установке выберите "Git Bash Here" в контекстном меню
3. После установки Git Bash доступен через контекстное меню или в меню Пуск

## Git Bash vs CMD

### Git Bash
- Плюсы:
  - Unix-подобные команды (`ls`, `grep`, `cat` и т.д.)
  - Удобен для скриптов
  - Поддержка SSH-ключей
  - Цветной вывод
- Минусы:
  - Дополнительная установка
  - Другая среда, чем стандартная Windows

### CMD (Command Prompt)
- Плюсы:
  - Уже есть в Windows
  - Знакомый интерфейс
- Минусы:
  - Меньше удобных команд
  - Менее удобен для Git

## Рекомендация

Для работы с Git удобнее Git Bash: он даёт больше возможностей и привычен многим разработчикам. Если нужна только базовая работа с Git, CMD тоже подойдёт.

Также можно использовать PowerShell (уже есть в Windows) или встроенный терминал в IDE (например, в IntelliJ IDEA).

## Разница между терминалами

### 1. **CMD (Command Prompt)**
- Что это: классическая командная строка Windows
- Команды: старые DOS-команды (`dir`, `copy`, `del`)
- Возможности: базовые операции с файлами и папками
- Git: работает, но без удобств
- Когда использовать: простые задачи, старые скрипты

### 2. **PowerShell**
- Что это: современная оболочка Windows (объектно-ориентированная)
- Команды: PowerShell cmdlets (`Get-ChildItem`, `Copy-Item`) + алиасы для старых команд
- Возможности: работа с объектами .NET, мощные скрипты, конвейеры
- Git: работает хорошо
- Когда использовать: автоматизация, администрирование Windows, современная разработка

### 3. **Git Bash**
- Что это: эмуляция Unix/Linux окружения для Windows
- Команды: Unix-команды (`ls`, `grep`, `cat`, `find`)
- Возможности: Unix-инструменты, bash-скрипты, SSH
- Git: встроен, удобен для Git-операций
- Когда использовать: работа с Git, кросс-платформенные проекты, привычка к Linux

### 4. **Терминал в IDE (IntelliJ IDEA)**
- Что это: встроенный терминал, который использует системную оболочку
- Команды: зависит от настроек (может быть CMD, PowerShell или Git Bash)
- Возможности: те же, что у выбранной оболочки, плюс интеграция с IDE
- Git: обычно хорошо интегрирован
- Когда использовать: удобно работать прямо в IDE, не переключаясь между окнами

## Сравнительная таблица

| Характеристика | CMD | PowerShell | Git Bash | Терминал IDE |
|----------------|-----|------------|----------|--------------|
| Установка | Встроен | Встроен (Win10+) | Нужна установка Git | Встроен |
| Команды | DOS | PowerShell + алиасы | Unix/Linux | Зависит от настроек |
| Git | Работает | Работает | Встроен, удобен | Зависит от настроек |
| Скрипты | .bat | .ps1 | .sh (bash) | Зависит от настроек |
| Кросс-платформа | Нет | Windows | Эмуляция Unix | Зависит от настроек |

## Рекомендации

- Для Java-разработки в IDEA: терминал IDE (удобно) или PowerShell (современный)
- Для Git: Git Bash (удобнее) или PowerShell
- Для общего использования: PowerShell (современный стандарт Windows)
- Для Linux-подобного опыта: Git Bash

В IDEA можно настроить терминал на использование Git Bash или PowerShell через Settings → Tools → Terminal → Shell path.
