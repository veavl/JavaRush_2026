# Настройка поведения AI ассистента в Cursor

## Где находятся настройки

1. **Откройте настройки:**
   - `Ctrl + ,` (или `File` → `Preferences` → `Settings`)
   - Или через меню: `File` → `Settings`

2. **Выберите раздел:**
   - В левой панели найдите **"Rules and Commands"** (Правила и команды)

## Типы правил

### User Rules (Пользовательские правила)
- **Применение:** Ко всем вашим проектам в Cursor
- **Когда использовать:** Для глобальных предпочтений поведения AI
- **Пример:** "Всегда только объясняй, не меняй код автоматически"

### Project Rules (Правила проекта)
- **Применение:** Только к текущему проекту
- **Когда использовать:** Для специфичных правил конкретного проекта
- **Пример:** "В этом проекте используй только Java 8 синтаксис"

## Как добавить правило

1. Нажмите кнопку **"+ Add Rule"** в нужном разделе (User Rules или Project Rules)
2. Введите текст правила
3. Сохраните

## Пример правила для "только объяснять"

```
По умолчанию ТОЛЬКО объясняй и отвечай на вопросы. 
НЕ изменяй код автоматически, если я явно не попрошу.
НЕ создавай файлы автоматически.
Изменяй код ТОЛЬКО если я явно скажу "Исправь", "Сделай", "Измени".
```

## Дополнительные опции

В разделе "Rules and Commands" также есть:

- **"Include CLAUDE.md in context"** — файлы CLAUDE.md и CLAUDE.local.md будут добавляться в контекст AI, когда это релевантно
- **"Import Claude Commands"** — загрузка команд из директорий .claude/commands вместе с .cursor/commands

## Альтернативный способ: файл .cursorrules

Можно также создать файл `.cursorrules` в корне проекта с правилами. Этот файл будет автоматически прочитан Cursor.

## Приоритет правил

1. **Project Rules** (правила проекта) — имеют наивысший приоритет
2. **User Rules** (пользовательские правила) — применяются, если нет правил проекта
3. **Файл .cursorrules** — альтернатива правилам проекта
